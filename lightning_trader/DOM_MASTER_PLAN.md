# LighTrade: 終極閃電下單 (DOM) 系統大師級規劃藍圖 (Master Plan)

這份規劃藍圖整合了競品分析、現有架構、UI/UX 設計規範以及高頻交易演算法（Trading Algo Patterns），旨在為 LighTrade 打造一個媲美甚至超越傳統券商（如永豐金）的「戰鬥級閃電下單」介面。

---

## 壹、 核心設計理念與願景

1. **極致的反應速度 (Ultra-Low Latency UI)**：DOM 的操作必須是肌肉記憶，不允許任何卡頓。
2. **視覺直覺化 (Intuitive Visualization)**：透過顏色、大字體、動態條列，讓大腦能在毫秒級別判斷多空局勢與自身部位。
3. **防呆與容錯 (Fail-safe Mechanisms)**：在追求極速的「戰鬥模式」下，依然保有嚴格的底層保護機制（如部位上限、防連點）。
4. **高度自訂性 (High Customizability)**：適配不同交易者的習慣，包含熱鍵、欄位與縮放。

---

## 貳、 UI/UX 與視覺層次深度優化

根據 `gemini_dom_style.txt` 與競品分析，我們必須徹底還原並現代化「紅藍對比」的經典配置。

### 1. 經典五欄式無縫佈局 (Classic 5-Col Seamless Layout)
* **結構**：`[刪買 / 買進 / 委買量] | 價格中樞 | [委賣量 / 賣出 / 刪賣]`。
* **網格與邊框**：使用 `border-collapse` 搭配深色邊框（如 `border-gray-700`），讓格線清晰但不刺眼，消除傳統 Excel 的生硬感。
* **緊湊行高**：極小化 `<td>` 的 padding，確保單一螢幕能顯示最大檔數（20-30檔），減少滑鼠滾動。

### 2. 精準的色彩工程 (Precision Color Coding)
* **價格欄 (Price)**：
  * 當前成交價（Current Price）整格高亮為**亮黃底深色字**（如 `bg-yellow-300 text-slate-900`）。
  * 價格數字根據昨收/平盤價變色：紅漲、綠跌、白平盤。
* **買賣力道區 (Bid/Ask)**：
  * **委買區 (左側)**：背景採用漸層或淡淡的粉紅色（如 `bg-red-950/30`），代表買盤支撐力。
  * **委賣區 (右側)**：背景採用淡淡的水藍/淺綠色（如 `bg-emerald-950/30`），代表賣盤壓力。
  * **委託量柱狀圖 (Volume Histogram)**：在委託量欄位內疊加半透明的進度條，直覺顯示該檔位的掛單厚度。

### 3. 字體排版 (Typography Hierarchy)
* **數字字型**：全面強制使用等寬數字 (`font-variant-numeric: tabular-nums`) 或 barlow，確保跳動時畫面不會左右抖動。
* **極端值標記**：當日最高價旁標註紅色小上標 `H`，最低價標註綠色小下標 `L`。

## 參、 高效能互動與操作邏輯

### 1. 全域熱鍵與無滑鼠操作 (Global Hotkeys)
* `Space (空白鍵)`：一鍵置中（Center to Last Price），將當前價格拉回畫面正中央。
* `Esc (取消鍵)`：緊急全刪單（Emergency Cancel All），立刻清除所有未成交委託。
* `方向鍵 (Up/Down)`：在 DOM 上下移動預選焦點。
* `Enter`：依照預選焦點直接送出市價或限價單。

### 2. 戰鬥模式與防呆機制 (Combat Mode & Safety)
* **戰鬥鎖頭 (Combat Toggle)**：明確的 UI 狀態顯示當前是否為「一鍵下單」模式。解鎖時單擊即送單，上鎖時需二次確認。
* **防連點機制 (Debounce)**：下單觸發後，該價格區塊或全域按鈕立即鎖定 0.5 秒（UI 顯示微暗），防止手震重複下單。
* **預設部位上限 (Max Position Limit)**：前端攔截與後端雙重檢查，送單前計算 `現有部位 + 新增委託`，超過設定值（如 10 口）直接阻擋並跳出警告。

### 3. 拖曳與快速改單 (Drag & Drop Modification)
* 允許將已掛單的標籤（顯示在買賣欄位外側）直接拖曳到新的價格層次，放開後自動發送 `update_order` API 進行改單。

## 肆、 進階交易演算法與策略整合 (Trading Algo Patterns)

這部分依賴 FastAPI 後端的 Memory State 與本地洗價能力。

### 1. 觸價單 (Market If Touched, MIT)
* **機制**：使用者在 DOM 上透過 `Shift + Click` 或特定按鈕，在遠離市價的地方掛出「觸價單」。
* **後端實作**：這些單不送到 Shioaji 交易所主機，而是存在 FastAPI 後端的 `SmartOrderMonitor`。當 Websocket 收到最新 Tick 跨越設定價位時，後端瞬間發送「市價單」。

### 2. 智慧移動停損 (Trailing Stop)
* **機制**：啟動後，系統自動追蹤獲利最高點（High Watermark）。
* **後端實作**：多單情境下，若市價創新高，停損線自動上移（例如：最高價減 10 點）。一旦價格回檔觸碰到移動停損線，立刻送出平倉市價單。此邏輯必須在後端獨立的非同步 Task 中執行，確保不受 UI 卡頓影響。

### 3. 閃電平倉與反向 (Quick Flatten & Reverse)
* **一鍵平倉**：讀取當前部位（如多單 5 口），點擊「平倉」時，自動送出相反方向的市價單（空單 5 口），並伴隨「全刪單」以確保乾淨出場。
* **一鍵反向**：讀取當前部位（如多單 5 口），點擊「反向」時，自動送出雙倍相反方向的市價單（空單 10 口），瞬間轉為空單 5 口。
